# Whack-a-mole

**Whack-a-mole** — небольшая аркадная игра на **Unity**, построенная вокруг классической механики «ударь крота». Игрок должен быстро нажимать на появляющихся кротов, зарабатывать очки, удерживать темп и улучшать свой результат по мере ускорения игры.

Проект ориентирован на короткие игровые сессии и подготовлен под публикацию в **WebGL / Yandex Games**: в нём есть локальное хранение рекорда, отправка результата в лидерборд и простая сцена с мгновенным рестартом после поражения.

## Что есть в проекте

* классическая аркадная механика Whack-a-mole;
* случайное появление кротов из набора лунок;
* подсчёт текущего счёта;
* сохранение рекорда;
* сохранение лучшего множителя;
* постепенное ускорение игры через множитель;
* анимации UI и объектов через LeanTween;
* таблица лидеров через Yandex Games;
* простой цикл: старт → серия попаданий → проигрыш → перезапуск сцены.

## Игровой цикл

1. После запуска сцены начинается короткая пауза перед стартом.
2. Кроты случайным образом появляются из доступных лунок.
3. Игрок кликом «бьёт» крота, получает очко и убирает его обратно.
4. По мере игры множитель растёт, а интервал появления уменьшается.
5. Если игрок не успевает, партия заканчивается на последнем оставшемся кроте.
6. Показывается экран проигрыша, после чего сцена перезапускается.

## Основные скрипты

### `MolesProvider.cs`

Главный контроллер игровой сессии:

* хранит массив активных кротов;
* запускает игровой цикл;
* случайно выбирает крота для появления;
* считает текущий счёт;
* обновляет рекорд;
* отправляет результат в лидерборд Yandex Games;
* обрабатывает завершение партии и перезапуск сцены.

### `Mole.cs`

Логика отдельного крота:

* поднимает крота из лунки;
* возвращает его обратно после попадания;
* воспроизводит ParticleSystem при ударе;
* вызывает событие попадания.

### `Multiplyer.cs`

Управляет ростом множителя:

* постепенно увеличивает множитель;
* уменьшает интервал между появлениями кротов;
* отображает текущее значение на UI;
* сохраняет лучший достигнутый множитель.

### `Records.cs`

Показывает сохранённый рекорд и лучший множитель.

### `CameraConstantWidth.cs`

Поддерживает адаптацию камеры под разные соотношения сторон экрана.

### `Pulse.cs`

Добавляет пульсирующую анимацию для UI-элементов.

### `Extra.cs`

Управляет показом дополнительного UI-элемента через `PlayerPrefs` и анимацию CanvasGroup.

## Данные и сохранения

Проект использует `PlayerPrefs` для локального хранения:

* рекорда (`record`);
* лучшего множителя (`mult`);
* флага дополнительного UI-элемента (`extra`).

Также при обновлении рекорда результат отправляется в лидерборд Yandex Games с идентификатором `lb`.

## Запуск проекта

### Требования

* Unity **2022.3.34f1**

### Локальный запуск

1. Открыть проект через Unity Hub.
2. Использовать версию **2022.3.34f1**.
3. Открыть сцену:

```text
Assets/Scenes/SampleScene.unity
```

4. Запустить сцену в Play Mode.
